# /note — note記事作成スキル

Silver Tree Coffee Roaster のブランドトーンに沿ったnote記事を作成する。
必ず `guidelines/02-brand-identity.md` を参照してから執筆すること。

## ターゲット読者（2軸）

| ターゲット | 人物像 | 求めていること |
|-----------|--------|--------------|
| **A：コーヒー好き** | スペシャルティに興味があるが選び方が分からない人 | 専門知識・選び方・ここで買う理由 |
| **B：コーヒー屋志望** | いつか小さなコーヒー屋をやりたい人 | 焙煎士のリアル・開業の実態・一人でやる仕事 |

## 記事タイプ（3種類）

### 🔵 知識系（ターゲットA向け）
コーヒーの専門知識を「選ぶ手助け」の形で届ける。
- 蒸らしの理由・産地の違い・焙煎度の選び方
- 「なぜここで買うのか」に自然につながる流れ
- 文字数目安：800〜1,500字

### 🟠 実況系（ターゲットB向け）
「一人でコーヒー屋をやるとはどういうことか」のリアルな記録。
- 焙煎の失敗・マルシェ準備・保健所への申請・軽バンDIY
- 数字・事実・本音を惜しまない（美化しない）
- 文字数目安：1,500〜3,000字

### 🟢 ストーリー系（両ターゲット向け）
Silver Tree の世界観・ブランドの核心を伝えるエッセイ。
- 4分の回復・手紙とコーヒー・旅するコーヒー屋の意味
- 体験を描写する（効能は説明しない）
- 文字数目安：1,000〜2,000字

## 執筆プロセス

### Step 1：テーマ確認
`$ARGUMENTS` からタイプとテーマを読み取る。
不明確な場合は以下を確認する：
- 記事タイプ（知識系 / 実況系 / ストーリー系）
- 主なテーマ・伝えたいこと
- 具体的なエピソード・事実があるか

### Step 2：構成を組む
記事タイプに応じた構成を作る。

**知識系の構成：**
```
書き出し（問いかけ or 体験描写）
↓
知識の本体（3〜5ポイント）
↓
Silver Tree との接点（押しつけない形で）
↓
締め（余韻を残す）
```

**実況系の構成：**
```
今日起きたこと（結論から入ってもいい）
↓
詳細・経緯・数字
↓
次にやること or 気づき
↓
読者へのひとこと（同じ道を歩む人への言葉）
```

**ストーリー系の構成：**
```
情景描写（場所・時間・感覚）
↓
核心に近づく
↓
Silver Tree が存在する理由
↓
余白で終わる（結論を言いすぎない）
```

### Step 3：執筆

**ブランドトーンの必須チェック（全タイプ共通）：**
- 短く書く。3行で言えることを5行にしない
- 体験を描写する。効能を説明しない
- 「美味しい」は使わない。香り・温度・音・時間で描く
- 「お得・限定・急ぐ」は絶対に使わない
- 押しつけない。読む人が自分で感じる余白を残す
- 末尾のCTAは入れてもよいが、一言・控えめに

**よい文章の例：**
```
✅「お湯を注ぐ4分間、何も考えていなかった。
   ただ、落ちていくコーヒーの香りだけがそこにあった。」

❌「このコーヒーを飲めばリラックスできます。ぜひ試してみてください！」
```

### Step 4：仕上げ確認

執筆後、以下を自己チェックする：
- [ ] 「疲れた人の隣に置けるか」で見たとき違和感がないか
- [ ] 使ってはいけない言葉が入っていないか
- [ ] 余白があるか（詰め込みすぎていないか）
- [ ] 最後の一文に余韻があるか

## テーマ候補（インスピレーション用）

**知識系：**
- 「蒸らしを30秒やる、たった一つの理由」
- 「White BlendとDark Blend、どちらを選ぶべきか」
- 「スペシャルティコーヒーとは何か、3行で説明する」

**実況系：**
- 「マルシェ出店の準備、全部見せます」
- 「焙煎を失敗した話」
- 「軽バンでコーヒー屋を始めるまでのこと」
- 「保健所に営業許可を取りに行った日」

**ストーリー系：**
- 「4分で、何かが変わる気がした」
- 「手紙を同梱し始めた理由」
- 「店を持たない、という選択」
