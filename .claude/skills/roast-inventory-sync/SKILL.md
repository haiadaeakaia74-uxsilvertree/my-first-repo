# /roast-inventory-sync — 焙煎・在庫整合確認

このSkillは、焙煎記録を在庫に反映するときのミスを防ぐために使う。
`roast-log` が焙煎記録とHTML生成を担当し、このSkillは在庫の加算・減算の確認を担当する。

## 原則

- 在庫は勝手に減算しない。まず現在値、使用量、反映済みかを確認する。
- `ops/inventory-log.md` の `## 最新在庫` と `## 焙煎ログ` の両方を見る。
- 同じ焙煎がすでに記録済みなら、二重減算しない。
- 銘柄名が似ている場合は合算しない。旧豆と新豆を分ける。
- SNS投稿文はこのSkillでは生成しない。

## 減算前チェック

1. 焙煎ログの生豆量を確認する。
2. 対象銘柄が在庫表のどの行か確認する。
3. 既に同じ日付・同じ豆・同じ使用量が `ops/inventory-log.md` にあるか確認する。
4. 減算前後の数値を計算する。
5. 不明点があればユーザーに確認する。

## 入荷時チェック

1. 納品書または写真から銘柄・数量を読む。
2. 既存銘柄に合算するか、新規銘柄として追加するか判断する。
3. 検証豆や産地違いは原則として新規行で分ける。
4. 総在庫を再計算する。
5. 入荷ログに「入荷確認日」を残す。

## 更新対象

- `ops/inventory-log.md`
- 必要に応じて `ops/inventory/index.html`
- 必要に応じて `ops/inventory/YYYY-MM-DD.html`

## 報告フォーマット

```text
在庫反映内容:
- 対象豆:
- 処理: 減算 / 入荷加算 / 確認のみ
- 変更前:
- 変更量:
- 変更後:
- 総在庫:
- 二重反映チェック:
```

## 禁止

- 生豆量が曖昧なまま減算する。
- 銘柄名が違う豆を合算する。
- すでに反映済みの焙煎をもう一度引く。
- SNS投稿文を自動生成する。
- X投稿、自動投稿、GitHub Actionsに触る。
