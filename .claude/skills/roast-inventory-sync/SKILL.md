# /roast-inventory-sync — 焙煎・在庫・SNS 連動スキル

焙煎完了後にやるべき「3つの後処理」を一括で行う。
毎回 `/roast-log` → `/inventory` を別々に呼んで手作業で在庫を更新する手間をなくす。

**処理内容：**
1. 焙煎プロファイルの記録（roast-profiles/）
2. 生豆在庫の自動減算（ops/inventory/）
3. 翌日SNS「焙煎しました」投稿文の生成

---

## Step 1：焙煎データの確認

`$ARGUMENTS` から読み取る。なければ以下を質問する。

### 必須情報
| 項目 | 例 |
|------|-----|
| 焙煎日 | 2026-05-10 |
| 豆の名前（産地・品種） | コロンビア スプレモ |
| 生豆重量 | 246.8g |
| 焙煎後重量 | 201g |
| 焙煎度 | 中煎り / 深煎り / 浅煎り |

### 追加情報（任意・あれば記録）
| 項目 | 例 |
|------|-----|
| 目標ドラム温度 | 220℃ |
| 投入温度 | 124℃ |
| 1ハゼ時刻 | 9分52秒 |
| 1ハゼ終了 | 9分55秒 |
| 排出時刻 | 13分49秒 |
| 時系列温度 | 0分:124℃/90℃、3分:162℃/138℃、... |
| 気づき・メモ | 自由記述 |

## Step 2：3つの処理を実行

### 処理①：焙煎プロファイル記録

`/roast-log` の仕様に従って以下を生成・保存する。

**温度データがある場合：**
- `roast-profiles/YYYY-MM-DD.html` を新規作成
  - Chart.js scatter デュアルY軸（ドラム温度・豆温度・ROR）
  - eventMarkerPlugin（1ハゼ・2ハゼ・排出の縦線マーカー）
  - インサイトセクション（ROR推移・歩留まり・前回比較）
- `roast-profiles/index.html` を更新
  - 新セッションカードを最上部に追加
  - 統計カード（セッション数・総バッチ数・平均歩留まり）を更新

**温度データがない場合：**
- `roast-logs/YYYY-MM-DD_[豆名]-batch[N].md` にマークダウンログを保存
- roast-profiles/index.html にテキストのみのカードを追加

---

### 処理②：在庫の自動減算

使用した豆の重量を在庫から引く。

**計算：**
```
消費量 = 生豆重量 ÷ 1000（g → kg）
新しい在庫 = 現在の在庫 - 消費量
```

**更新ファイル：`ops/inventory/index.html`**

該当豆の在庫数を更新する：
- 豆の在庫量（kg）を更新
- 総在庫量を再計算
- 最終更新日を当日に変更
- トレンドグラフのデータに当日の数値を追加

在庫が **2kg を下回った場合** は補充アラートを出力する：
```
⚠️ [豆名] の在庫が2kgを下回りました（残：___kg）
補充推奨量：___kg
```

---

### 処理③：翌日 X 投稿文の生成

「焙煎しました」系の柱②（リアル実況）投稿文を生成する。
`guidelines/05-sns-operations.md` のルールに従う。

**テンプレート（選択肢を2案）：**

```
案A（豆の特徴を前面に）：
[豆名]を焙煎した。

[焙煎で気づいたこと・豆の香りや特徴 1〜2文]

[White Blend / Dark Blend / シングルオリジン]として使う。

#自家焙煎 #スペシャルティコーヒー

---

案B（作業のリアル実況）：
今日の焙煎、[バッチ数]バッチ。

[工程のリアルな描写 1〜2文]
歩留まり[XX]%。[良かった点 or 課題を一言]

#旅するコーヒー屋 #SilverTreeCoffee
```
