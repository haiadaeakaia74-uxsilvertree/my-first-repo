# /marketing — 投稿前判断・違和感分解

## 目的

このSkillは、投稿文を大量に作るためではなく、投稿前の判断を深めるために使う。

文章の先には、必ず見えない読者がいる。投稿は作業報告ではなく、読者にどう行動してもらい、その先にどんな価値を渡すかから逆算する。

## タスク

$ARGUMENTS

## 最初にやること

投稿案を出す前に、まず違和感を分解する。

1. 何に違和感があるのか。
   - 単調
   - AIっぽい
   - 自分目線が強い
   - 売り込みに見える
   - 読者が動く理由がない
   - 今出す意味が弱い
2. 誰に向けた投稿か。
   - まだ飲んだことがない人
   - 応援するか迷っている人
   - 出店で会う可能性がある人
   - BASEやCAMPFIREを見る直前の人
3. 相手にどう動いてほしいか。
   - 飲んでみたい
   - 応援したい
   - プロフィールやCAMPFIREを見たい
   - 出店で会いたい
   - コメントしたい
4. 行動した先にどんな価値を渡せるか。
   - 新鮮なコーヒーを飲める
   - 旅するコーヒー屋を一緒に育てられる
   - 応援が実際の一杯になって返ってくる
   - 自分に合うコーヒーの選び方がわかる
5. 今この投稿を出す意味があるか。
   - 今日の活動とつながっているか
   - クラファンや出店準備の流れとつながっているか
   - 読者にとって今知る理由があるか

## 判断チェック

投稿文を見るときは、最低限これを確認する。

- 自分の作業説明だけで終わっていないか。
- 設備、道具、数字の話が、読者の価値につながっているか。
- 「だから何？」で止まらないか。
- 読者が次に何をすればよいか見えるか。
- 売り込みが強く、応援したい気持ちを削っていないか。
- 気分だけのポエムになっていないか。
- クイズ、比較、実例、写真、設計図など、反応しやすい形にできるか。

## 画像・デザインの分担

Silver TreeのInstagram用完成画像や世界観のある1枚絵は、ユーザーがCanva GPT等で作る。
このSkillでは、完成画像を作らない。

Codexが担当すること:

- 投稿やスライドの構成を作る
- 画像に載せる文章を整理する
- 読者に見せる順番を決める
- PDF化、PPTX化、HTML化の下準備をする
- ファイル管理やリンク追加をする

Codexが担当しないこと:

- Instagram用の完成画像デザイン
- 高品質なビジュアル生成
- 世界観のある1枚絵
- 余白、色、構図の美的な最終判断
- Canva風のデザイン完成品

画像や世界観に関わる判断でブレが出そうな場合は、先に確認する。

## 出力の順番

まず判断を返し、必要な場合だけ最後に投稿案を1案出す。

1. 違和感の正体
2. 想定読者
3. 動いてほしい行動
4. 読者に渡す価値
5. 今出す意味
6. 修正方針
7. 投稿案（必要な場合のみ1案）

## 書き換えの考え方

設備や準備の話は、単体では弱い。必ず「誰のために」「何を届けるために」へつなげる。

弱い順番:

```text
水タンクを用意しました。
排水バケツも必要です。
衛生的に営業します。
```

良い順番:

```text
外でも安心して一杯を渡したい。
そのために水の使い方と片付け方を先に決める。
味の話をする前に、飲みに来てもらえる場所を整える。
```

## 禁止

- いきなり投稿文を大量生成しない。
- 役職なりきりだけで判断しない。
- 自動投稿前提にしない。
- 完成画像デザインまでCodexで引き受けない。
- 違和感やブレが出そうなまま押し切らない。
- `x-post-log.md`、`x_poster.py`、GitHub Actions、`.github/workflows/` は触らない。
- 指示された範囲を超えて投稿予約や投稿実行をしない。

## X投稿の心を動かす基準

文章の先には必ず見えない読者がいる。投稿文は作業報告ではなく、読者にどう行動してもらい、その先にどんな価値を渡すかから逆算する。

投稿前に必ず確認する。

1. 誰が読むのか。
   - まだ飲んだことがない人
   - 応援するか迷っている人
   - いつか出店に来てくれる人
2. その人にどう動いてほしいのか。
   - 飲んでみたい
   - 応援したい
   - 出店で会いたい
   - プロフィールやCAMPFIREを見たい
3. 行動した先にどんな価値を提供できるのか。
   - 新鮮なコーヒーを飲める
   - 旅するコーヒー屋を一緒に育てられる
   - 応援した活動が、実際の一杯になって返ってくる
4. だから今何を準備しているのか。
   - 軽バン、露店設備、水まわり、焙煎、商品設計はすべて手段として書く。

避ける書き方:

- 水タンク、採寸、棚、消毒用品などの設備説明だけで終わる。
- 「〇〇しました」「〇〇が必要です」だけで、読者の行動や価値が見えない。
- ポエム調で気分だけを書く。

良い順番:

```text
応援してくれた人に、新鮮な一杯を届けたい。
そのために町へ行ける露店活動を作る。
だから軽バンや露店設備を整える。
```

## 注意

実際の X への投稿実行は `/x-post` を使用する（副作用のある操作のため分離）。

X投稿予定を編集した場合は、ローカルの `x-post-log.md` 修正だけで完了にしない。
自動投稿は GitHub の `main` を読むため、必要な投稿関連ファイルだけをコミット・pushし、`origin/main` に最新本文が入ったことを確認してから完了報告する。
