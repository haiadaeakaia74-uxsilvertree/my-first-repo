# /base-product — BASE商品ページ最適化スキル

Silver Tree Coffee Roaster のBASEショップ商品ページを、ブランドトーンに沿って作成・改善する。
`guidelines/02-brand-identity.md` と `guidelines/03-product-guide.md` を必ず参照すること。

---

## Step 1：商品情報の確認

`$ARGUMENTS` から商品名を読み取る。不足情報は質問する。

| 項目 | 必須 | 例 |
|------|------|-----|
| 商品名 | ○ | White Blend 100g |
| 焙煎度・特徴 | △ | 浅煎り・ハニー・アーモンドの香り |
| ターゲット | △ | 朝に飲む人 / コーヒー初心者 |
| 価格 | △ | 1,200円 |
| セット内容 | △ | 豆のみ / 豆 + 手紙 |

---

## Step 2：3点セットを生成

### ① 商品タイトル（60字以内）

BASE検索に引っかかりやすく、Silver Treeらしいタイトルを3案出す。

**構造：**
```
[商品名] ／ [一言キャッチ] ／ [容量・形式]
```

**例：**
```
✅ White Blend 100g｜朝の呼吸を整えるコーヒー｜Silver Tree Coffee Roaster
✅ 【自家焙煎】Dark Blend 豆 100g｜深夜の静けさに寄り添うコーヒー
```

---

### ② キャッチコピー（30字以内）

商品の核心を一文で表す。「体験」を描写する。

**NG例（使わない）：**
```
❌「美味しいコーヒーをお届け！」
❌「厳選した豆を使用しています」
❌「お得なセットです！」
```

**OK例：**
```
✅「4分で、呼吸が変わる。」
✅「香りから始まる、朝の余白。」
✅「疲れた夜に、そっと置くコーヒー。」
```

---

### ③ 商品説明文（200〜400字）

**構造：**
```
[体験の描写（2〜3文）]

[商品の特徴（箇条書き or 文章）]

[どんな人に合うか（押しつけない形で）]

[同梱物・配送について（1〜2文）]
```

**文体ルール：**
- 「美味しい」は使わない → 香り・温度・余韻・時間で描写する
- 「ぜひ」「おすすめ」「ご注文はこちら」は使わない
- 余白のある文章。詰め込まない
- 最後は余韻で終わる

**例文（White Blend）：**
```
朝、ドリッパーにお湯を注ぐ4分間。
その静けさの中に、ハニーのような甘みと、柔らかい香りが広がります。

Silver Tree の White Blend は、香川の小さな焙煎所で一釜ずつ焙煎しています。
浅めの焙煎で、酸味よりも甘みを引き出しました。

コーヒーを飲み始めたばかりの人にも、
毎朝コーヒーを淹れる人にも。

豆のまま（粗挽き対応可）でお届けします。
手紙を1枚、一緒に入れています。
```
