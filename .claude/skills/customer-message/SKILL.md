# /customer-message — 顧客メッセージ返信スキル

Silver Tree Coffee Roaster の顧客とのコミュニケーションをサポートする。
`guidelines/04-customer-communication.md` と `guidelines/02-brand-identity.md` を必ず参照すること。

---

## Step 1：状況タイプの確認

`$ARGUMENTS` から状況を読み取る。なければ以下から選んでもらう。

| タイプ | 場面 |
|--------|------|
| A：感謝 | 「購入してくれた人へのお礼」「来店・再来のお礼」 |
| B：問い合わせ回答 | 「豆の選び方を聞かれた」「配送について」「EC注文の確認」 |
| C：来なくなった人へ | 「久しぶりのお客さんへの声かけ」「再アプローチ」 |
| D：クレーム・不満対応 | 「商品に不満があった」「対応へのお詫び」 |
| E：イベント案内 | 「出店告知を個人的に送る」「常連さんへの先行案内」 |

---

## Step 2：追加情報の確認

状況に応じて以下を確認する：

- 相手との関係（初めての人 / 常連 / 知り合い）
- 媒体（LINE / Instagram DM / BASE メッセージ）
- 特記事項（名前・購入商品・エピソードなど）

---

## Step 3：返信文生成

### Silver Treeの文体原則

- **一対一の手紙を書く感覚**で書く
- 丁寧だが距離を置かない（「です・ます」ベース）
- 短く・余白がある（200字以内を目安）
- 感謝より「気にかけている」が伝わる文章
- 「ぜひ」「よろしくお願いします」の多用を避ける

---

### タイプ別ガイドライン

**A：感謝メッセージ**
```
構造：
受け取ってもらえた喜び（1文）
→ 商品・コーヒーについて一言（1〜2文）
→ 余韻で締める（1文）

NG：「またよろしくお願いします！」「ぜひまたご購入ください」
OK：「またいつでも」「コーヒーが、その日の一部になれたら」
```

**B：問い合わせ回答**
```
構造：
質問への直接の答え（最初に）
→ 補足説明（必要なら）
→ 「何かあればいつでも」で締める

注意：専門用語を使うときは一言添える
```

**C：久しぶりの人へ**
```
構造：
思い出した・気にかけていることを一言（押しつけない）
→ 近況や季節について（1〜2文）
→ 来ることを強要しない締め

NG：「最近どうしましたか？」「ぜひまた来てください！」
OK：「春になってきたので、ふと思い出しました」
```

**D：クレーム・お詫び**
```
構造：
まず受け取る（言い訳より先に）
→ 事実確認・状況説明（短く・正直に）
→ 対応策（具体的に）
→ 締め（「ご不便をおかけしました」より「知らせてくれてありがとう」）

注意：謝りすぎない・感情的にならない・1時間は冷却してから返信
```

**E：イベント案内**
```
構造：
個人的な声かけ感覚で（告知文を貼りつけない）
→ 場所・日時（シンプルに）
→ 来なくてもいい、という余白

NG：「ぜひいらしてください！お待ちしています！」
OK：「もし気が向いたら、顔を出してもらえたら嬉しいです」
```
