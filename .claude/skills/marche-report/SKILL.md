# /marche-report — マルシェ事後レポートスキル

マルシェ・イベント終了後の振り返りを **構造化して記録** する。
次回出店の改善につなげ、財務・戦略ファイルを常に最新に保つ。

---

## Step 1：基本情報の確認

`$ARGUMENTS` から読み取る。なければ質問する。

| 項目 | 必須 | 例 |
|------|------|-----|
| イベント名 | ○ | ハピマル / 海岸寺マルシェ |
| 出店日 | ○ | 2026/6/28 |
| 売上（税込） | ○ | 12,500円 |
| 来客数（概算） | △ | 25名 |
| 購入者数 | △ | 8名 |
| LINE登録数 | △ | 3名 |
| BASE誘導数 | △ | 2名（QRコード渡し） |
| 天気・環境 | △ | 晴れ・人通り多め |
| 良かったこと | △ | 自由記述 |
| 改善したいこと | △ | 自由記述 |

---

## Step 2：採算計算

以下を自動計算する：

```
売上：[入力値]円
出店費：[入力値 or guidelines/06 から参照]円
交通費・諸経費：[入力値 or 概算]円
---------------------------------
純利益：売上 - 出店費 - 諸経費 = ___円
損益分岐点：(出店費 + 諸経費) ÷ 客単価 = ___名
実際の客単価：売上 ÷ 購入者数 = ___円
```

`guidelines/07-financial-management.md` の採算計算式を参照する。

```markdown
| 来客数 | ___名 |
| 購入者数 | ___名（転換率 ___％） |

| BASE誘導 | ___名 |
| 客単価 | ___円 |

## 採算
- 出店費：___円
- 純利益：___円
- 損益分岐点：___名（実績 ___名）

## 今日よかったこと
- （自由記述から整理）

## 次回改善したいこと
- （自由記述から整理）

## 次回への引き継ぎ
- （ファイルへの記録・SNS投稿・フォローアップなど）
```
