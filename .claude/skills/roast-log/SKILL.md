# /roast-log — 焙煎記録スキル

Silver Tree Coffee Roaster の焙煎データを記録・整理・振り返りできる形にまとめる。

---

## Step 1：焙煎データの収集

`$ARGUMENTS` から豆名・日付を読み取る。以下の情報を確認する（わかる範囲でOK）。

### 基本情報
| 項目 | 例 |
|------|-----|
| 焙煎日 | 2026-04-17 |
| 豆の産地・名称 | エチオピア ナチュラル / White Blend 用 |
| 生豆重量 | 300g |
| 焙煎後重量 | 255g |
| 焙煎度 | 浅煎り / 中煎り / 深煎り |

### 工程データ（わかる範囲で）
| 項目 | 例 |
|------|-----|
| 投入温度 | 160℃ |
| 1ハゼ開始 | 9分30秒 / 185℃ |
| 2ハゼ開始 | 13分 / 205℃（深煎りの場合） |
| 排出温度 | 195℃ |
| 総焙煎時間 | 12分30秒 |

### 結果・評価
| 項目 | 例 |
|------|-----|
| 外観 | 色ムラなし / ハネ多め |
| 香り（生豆時） | フルーティー・蜂蜜 |
| 香り（焙煎後） | 甘さが増した・焦げ感なし |
| 次回への改善点 | 投入温度を5℃上げる |
| 総合評価 | ★★★★☆ |

|------|-----|

## 保存先

### マークダウンログ（数値データ・テキスト記録）

```
roast-logs/YYYY-MM-DD_[豆名]-[バッチ番号].md
```

例：`roast-logs/2026-04-22_brazil-bellavista-batch1.md`

## Step 4：焙煎プロファイルHTML生成（温度ログがある場合）

温度プロファイルデータ（ドラム温度・豆温度の時系列）が揃っている場合は、
HTMLビジュアライゼーションを自動生成してプロファイルフォルダに保存する。

### HTML仕様（標準フォーマット）

| 要素 | 内容 |
|------|------|
| グラフ | バッチ個別グラフ（1バッチ＝1グラフ）×全バッチ |
| 3系列 | ドラム温度（緑実線・左軸）、豆温度（茶実線・左軸）、ROR（オレンジ破線・右軸） |
| マーカー | 1ハゼ・2ハゼ・排出を垂直破線＋ラベルで表示 |
| データテーブル | 各バッチの全計測値をグラフ直下に表示 |
| サマリーカード | 排出豆温・歩留まりを3バッチ並列で表示 |
| 分析インサイト | 5項目の考察（ROR推移・歩留まり比較・異常値など） |
| スタイル | Silver Tree ブランドカラー（#2D4A3E / #6B4226 / #F8F4EE） |

### HTMLファイル保存先

```
roast-profiles/YYYY-MM-DD.html
```

例：`roast-profiles/2026-04-22.html`

### index.html の更新

`roast-profiles/index.html` に新セッションのカードを追加し、
統計カード（セッション数・総バッチ数・平均歩留まり）を更新する。

参照フォーマット：`roast-profiles/2026-04-22.html`（初回作成時のマスター）
